# Project Documentation Overview

This document serves as a central directory for all technical and project-related documentation for the React Shopping Cart application. It is designed to provide a comprehensive resource for developers, stakeholders, and new team members, covering various aspects from project vision to coding standards and deployment.

All detailed documentation files are located in the `docs/` directory.

---

## Documentation Categories:

### 1. Inicio y Alcance (Initiation & Scope)

*   **[Project Vision](docs/project-vision.md):** Defines the project's purpose, objectives, and scope.
*   **[Glossary](docs/glossary.md):** Provides definitions for key technical and business terms used in the project.

### 2. Requerimientos (Requirements)

*   **[Requirements Document (FR / NFR)](docs/requirements.md):** Details functional and non-functional requirements.
*   **[Use Cases](docs/use-cases.md):** Describes key user interactions and system behaviors.

### 3. Arquitectura y Diseño (Architecture & Design)

*   **[Software Architecture](docs/software-architecture.md):** Describes the overall architectural style (Feature-Sliced Design), structure, and key decisions.
*   **[Design Patterns](docs/design-patterns.md):** Explains the design patterns applied in the project (e.g., Container/Presentation, Optimistic UI).
*   **[Design System](docs/design-system.md):** Outlines the visual design elements, components, and guidelines.

### 4. Desarrollo e Implementación (Development & Implementation)

*   **[Getting Started](docs/getting-started.md):** Instructions for setting up the development environment.
*   **[Coding Standards](docs/coding-standards.md):** Defines coding conventions, naming, and best practices.
*   **[API Integration](docs/api-integration.md):** Details the interaction with external APIs, endpoints, and data models.
*   **[Project Exercises](docs/project-exercises.md):** Practical exercises for new developers to onboard quickly.

### 5. Calidad, Seguridad y Despliegue (Quality, Security & Deployment)

*   **[Testing Strategy](docs/testing-strategy.md):** Defines the approach to unit, integration, and E2E testing.
*   **[Deployment](docs/deployment.md):** Describes the build and deployment process.
*   **[Security](docs/security.md):** Outlines security considerations and measures.

### 6. Cierre y Mantenimiento (Closure & Maintenance)

*   **[Maintenance Plan](docs/maintenance-plan.md):** Plan for ongoing maintenance, monitoring, and updates.
*   **[Scrum Artifacts](docs/scrum-artifacts.md):** Explains how Scrum roles, events, and artifacts are utilized.

---

This documentation aims to be a living resource, continuously updated to reflect the current state and evolution of the project.
